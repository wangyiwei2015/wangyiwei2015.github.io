常用国内加速站点

  2022-04-28   教程   配置环境 阅读时长: 3分

由于国内特殊网络环境，常常会遇到某些地址不可访问或速度缓慢；本页面整理了一些常用的替代站点。

目录：| GitHub | APT | PyPI | npm | RubyGems | Homebrew | Flutter | Maven | Docker | NTP | DNS |


GitHub

网页

油猴脚本： GitHub增强

Clone

将原URL

git clone https://github.com/{user}/{repo}

替换为

git clone https://gitclone.com/github.com/{user}/{repo}

推荐使用上述脚本，在网页上可以看到多个加速地址选项。


APT

适用于使用 apt 包管理的发行版。

用任意文本编辑器打开 sources.list

sudo nano /etc/apt/sources.list

或者先安装 apt-spy 自动寻找最佳。

可用的镜像站：

  • http://mirrors.163.com
  • http://mirrors.aliyun.com
  • https://mirrors.huaweicloud.com
  • https://mirrors.ustc.edu.cn
  • https://mirrors.tuna.tsinghua.edu.cn
  • https://mirror.sjtu.edu.cn

Debian

# 原始信息
deb http://deb.debian.org/debian/ 后面照抄
# deb-src http://deb.debian.org/debian/ 后面照抄

# 修改为
deb 镜像站/debian/ 后面照抄
# deb-src 镜像站/debian/ 后面照抄

Ubuntu

同上，debian 换成 ubuntu ；也可以进入桌面使用图形化的设置自动选择最佳地址。

Raspberry Pi OS （前 Raspbian）

除了 /etc/apt/sources.list 之外，还需要编辑 /etc/apt/sources.list.d/raspi.list 文件。

方法同上，可能有三种：debian , raspberrypi , raspbian/raspbian ，视情况照抄即可。

对于手动编辑文件的方式，完成后更新一下： sudo apt update 。


PyPI

可用的镜像站：

  • 清华源 https://pypi.tuna.tsinghua.edu.cn/simple
  • 阿里源 https://mirrors.aliyun.com/pypi/simple
  • 腾讯源 http://mirrors.cloud.tencent.com/pypi/simple
  • 豆瓣源 http://pypi.douban.com/simple
  • 中科大 http://pypi.mirrors.ustc.edu.cn/simple

单次换源

pip install 包名 -i 源地址

永久设置

# 设置
pip config set global.index-url 源地址
# 还原
pip config unset global.index-url

npm

可用的镜像站：

  • 淘宝源 https://registry.npm.taobao.org
  • cnpmjs http://r.cnpmjs.org/

单次换源

npm install -选项 包名 --registry=源地址

永久设置

# 设置
npm config set registry 源地址
# 还原
npm config set registry https://registry.npmjs.org

Ruby Gems

gem sources --remove https://rubygems.org/
gem sources --add https://ruby.taobao.org/
gem sources --add http://gems.ruby-china.com/
#更新缓存
gem sources -u

Homebrew

加速 GitHub

cd "$(brew --repo)"
git remote set-url origin git://mirrors.ustc.edu.cn/brew.git
cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
git remote set-url origin git://mirrors.ustc.edu.cn/homebrew-core.git
# 或者！
cd "$(brew --repo)"
git remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/brew.git
cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
git remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew-core.git
# 或者！
cd "$(brew --repo)"
git remote set-url origin https://mirrors.aliyun.com/homebrew/brew.git
cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
git remote set-url origin https://mirrors.aliyun.com/homebrew/homebrew-core.git
# 注：官方仓库在 github.com

加速 bottles

新版 macOS 默认使用 zsh ，修改 ~/.zshrc 文件；其它版本修改对应的 profile 文件。

文件末尾加入以下内容

export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
# 或者！
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles

然后可能需要重新打开终端使其生效。


Flutter

以下默认替换为 flutter-io 的源；可按需选用清华源：

  • pub - https://mirrors.tuna.tsinghua.edu.cn/dart-pub
  • storage - https://mirrors.tuna.tsinghua.edu.cn/flutter

修改环境变量。

Windows

开始 -> 搜索 env 或找到 编辑系统环境变量 -> 高级 -> 环境变量

选择用户或系统皆可。

编辑或新建 PUB_HOSTED_URL ，值为 https://pub.flutter-io.cn 。

编辑或新建 FLUTTER_STORAGE_BASE_URL ，值为 https://storage.flutter-io.cn 。

macOS & Linux

新版 macOS 默认使用 zsh ，修改 ~/.zshrc 文件；其它版本修改对应的 profile 文件。

文件末尾加入以下内容

export PUB_HOSTED_URL=https://pub.flutter-io.cn
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn

然后可能需要重新打开终端使其生效；运行 flutter doctor 检查。


Maven

找到 maven 的配置文件；一般位于 apache-maven-版本\conf\settings.xml 。

对于 JetBrains IDEA IDE ，找到 设置 -> 构建/执行/部署 -> 用户配置文件 的目录。

替换 mirror 相关内容：

<mirrors>
    <mirror>
        <id>alimaven</id>
        <mirrorOf>central</mirrorOf>
        <name>aliyun maven</name>
        <url>http://maven.aliyun.com/nexus/content/repositories/central/</url>
    </mirror>
    <mirror>
        <id>alimaven</id>
        <name>aliyun maven</name>
        <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
        <mirrorOf>central</mirrorOf>
    </mirror>
    <!-- 中央仓库在中国的镜像 -->
    <mirror>
        <id>maven.net.cn</id>
        <name>oneof the central mirrors in china</name>
        <url>http://maven.net.cn/content/groups/public/</url>
        <mirrorOf>central</mirrorOf>
    </mirror>
</mirrors>

Docker

可用的镜像站：

  • Docker中国 https://registry.docker-cn.com
  • 网易 http://hub-mirror.c.163.com
  • 中科大 https://docker.mirrors.ustc.edu.cn

单次换源

docker pull 源地址/myname/myrepo:mytag

永久设置

修改 /etc/docker/daemon.json

{
  "registry-mirrors": ["源地址"，可以多个]
}

NTP

  • 中国授时 cn.ntp.org.cn
  • 国家授时 ntp.ntsc.ac.cn
  • ntp.org [0-3]?.cn.pool.ntp.org （例 cn.pool.ntp.org）
  • 阿里云 ntp[1-7].aliyun.com （例 ntp7.aliyun.com）
  • 教育网 edu.ntp.org.cn
  • 国内高校 s[1-9][a-z].time.edu.cn （例 北大 s1c.time.edu.cn）
[!] 关于 Home Assistant 安装的一个坑

第一次开机会同步时间，默认使用 Google 服务器，于是同步失败。此后无报错但是 Docker 初始化失败，系统不能启动。解决方法是在烧录系统到 SD 卡后挂载 FAT 分区，创建 CONFIG/timesyncd.conf ，写入以下内容

[Time]
NTP=选一个上述NTP服务器
FallbackNTP=再选一个NTP服务器

再上电开机。
